# Course 3 - Module 2 - CSS Overview & HTML5 Elements

## Key Concepts
- CSS maintains consistent styling across all pages — best managed via external stylesheets
- CSS frameworks speed up UI development — two types:
  - Utility-first (e.g. Tailwind) — gives you utility classes, you build custom styles
  - Component-based (e.g. Bootstrap) — gives you pre-styled ready-made components
- Vanilla CSS = writing styles from scratch without any framework
- HTML5 elements form the structural and functional foundation of a webpage
- `<input>` tag handles all user data entry — text, password, checkbox, radio, etc.

## New Terms
- CSS Framework: pre-built collection of styles/components that speeds up development
- Utility-first Framework: gives atomic utility classes — you compose styles yourself
- Component Framework: gives complete pre-styled UI components out of the box
- Vanilla CSS: pure CSS with no framework — full control, more code
- `<input>` tag: HTML5 element for capturing user input in forms

## Summary
CSS keeps a website visually consistent and is most efficiently managed through external
stylesheets. Frameworks (utility-first or component-based) speed up styling significantly.
Vanilla CSS gives full control but requires more effort. HTML5 elements structure the page
and the input tag handles all forms of user data entry.
