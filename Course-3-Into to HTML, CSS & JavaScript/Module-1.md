# Course 3 - Module 1 - HTML Overview

## Key Concepts
- HTML uses tags to define structure and content of a webpage
- HTML DOM Tree = hierarchical tree of all elements on a page
- HTML5 introduced APIs for animation, audio, video — major UX upgrade
- Scripting adds interactivity but can be disabled by users — don't fully rely on it
- CSS handles styling — fluid layout adapts to screen size, fixed layout stays constant
- Browser support for HTML5 features can be checked via support tables or JavaScript

## New Terms
- HTML DOM Tree: hierarchical representation of webpage elements — parent/child structure
- HTML5 Sandbox: security feature that restricts what iframe content can do
- Fluid Layout: CSS layout that scales with screen size (responsive)
- Fixed Layout: CSS layout with hardcoded dimensions regardless of screen size

## Summary
HTML provides the structural backbone of every webpage using tags and the DOM tree.
HTML5 extended this with powerful APIs for media and animation. CSS sits on top to
handle styling, offering fluid or fixed layouts. Scripting adds interactivity but
browser compatibility and security (sandboxing) must always be considered.
