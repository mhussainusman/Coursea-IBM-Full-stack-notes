# Course 1 - Module 3 - Basics of Programming

## Key Concepts
- Interpreted languages run directly through an interpreter line by line — no compilation
- Compiled languages translate entire source code into an executable file first
- High-level languages: query, structured, OOP — closer to human language
- Low-level languages: assembly — closer to machine code
- Flowcharts = visual representation of an algorithm
- Pseudocode = plain language step-by-step logic before actual coding
- Identifiers: names given to variables, constants, functions in code
- Functions: reusable code blocks that perform a specific task
- OOP focuses on objects rather than functions — you already know this from C++

## New Terms
- Interpreter: executes source code directly without compiling
- Compiler: translates entire source code into machine-executable file
- Identifier: any name that references a component in your program

## Summary
Programming languages are either interpreted (run line by line) or compiled (translated
first then executed). Code is organized using functions and OOP principles. Before
writing actual code developers plan logic using flowcharts and pseudocode.
