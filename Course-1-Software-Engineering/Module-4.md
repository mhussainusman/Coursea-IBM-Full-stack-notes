# Course 1 - Module 4 - Software Architecture, Design and Pattern

## Key Concepts
- Software architecture is the blueprint of a system — defines structure before coding starts
- Structured design breaks complex problems into smaller manageable components
- UML diagrams visually model software — class diagrams, state diagrams, interaction diagrams
- Architectural patterns are reusable solutions to common design problems
- Environments: Development → Testing → Staging → Production
- Production environment is live — needs load handling, security and scalability

## New Terms
- UML: Unified Modeling Language — visual diagrams to plan and represent software structure
- Architectural Pattern: reusable template for solving a common system design problem
- SOA: Service-Oriented Architecture — loosely coupled services communicating over a network
- Production Environment: live environment where real users access the software

## Summary
Software architecture defines how a system is structured before a single line of code
is written. UML diagrams help visualize that structure. Architectural patterns like
SOA provide proven solutions to common design challenges. Understanding different
environments — especially production — is critical for deploying real software.
